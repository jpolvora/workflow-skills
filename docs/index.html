Workflows

Choose the right workflow orchestrator for your task's scope and speed requirements. Both share shared/config.json and gates.md (dual-mode). Session model on every transition; switch via Pause → Cursor → Resume (no --model / --model-chain). Standard step dispatch: STEP-DISPATCH.md (not used as lite step numbers).

Standard

spec-to-pr

Full multi-stage pipeline (Steps 0–13)

A comprehensive Finite State Machine (FSM) orchestrator designed for end-to-end software lifecycle delivery. Best for complex tasks starting from feature descriptions or issues.

  • Collaborative Spec Brainstorming (Step 0)
  • Architectural Review & Interview (Steps 1–2)
  • DAG Task Decomposition (Step 3)
  • Parallel or Multi-layered coding (Steps 5, 10)
  • Pre-PR validation & goal-fix-pr loops (Steps 11–13)
  • Session model + phase soft tips (Coder/Reviewer); Pause to switch
View spec-to-pr workflow
Lite

spec-to-pr-lite

Fast sequential pipeline (Steps 1–5)

A streamlined, fast alternative designed to run planning, sequential coding, code review, and shipping in a direct loop. Requires an existing specification.

  • Planning and brainstorm from existing specification
  • Direct sequential coding (no complex DAG required)
  • Quick local code review & automatic fix cycles
  • Fast consolidation and PR shipping
  • Zero setup/config duplication in dual-mode
  • Session model banner; Pause → Cursor → Resume to switch
View spec-to-pr-lite workflow

Installation packages

The interactive installer (npx --yes github:jpolvora/workflow-skills) supports package shortcuts and skill-by-skill selection. Membership and install-time dependencies are defined in bin/skill-dependencies.json (update the map whenever the installer graph changes).

Full / Workflows installs also ensure the shared/ hub and create-if-missing root seeds (.cursorrules → AGENTS.md, CHANGELOG.md stub) — never overwrite existing files. Packaged .agents/AGENTS.md includes portable External dependencies so consumers are not dead-ended when the root hub omits that section.

Full package (f)

Full package — selects every installable top-level skill and installs the shared/ config/docs hub.

Workflows package (w)

Workflows package — 26 skills (orchestrators, pipeline, providers, harness, promoted utilities) plus the shared/ hub. Does not force Extra-only skills.

Includes: spec-to-pr, spec-to-pr-lite, 00-write-spec, 01-write-plan, 02-interview, 03-plan-to-tasks, 04-implement-tasks, 05-verify-plan, … (+18)

Extra package (e)

Extra package — 9 standalone review/design/meta skills. Does not install workflow orchestrators or the hub by default.

Includes: security-review, dotnet-security-performance-review, tdd-sdd-ddd-reviewer, domain-review, multi-domain-review, secrets-leak-review, mobile-first-design, taste-skill, write-a-skill

Individual selection

Toggle skills by number. Selecting a skill also selects its transitive install dependencies from the map. Deselecting a skill does not cascade-deselect dependencies.

Install

Canonical form: npx --yes github:jpolvora/workflow-skills (never @latest / @main on github:). Consumer-owned under .agents/skills/shared/: config.json, stack.md, MEMORY.md, memory/ — preserved on update. Full / Workflows also seed create-if-missing root .cursorrules and CHANGELOG.md (never overwrite).

Interactive Menu

Open the visual menu to select packages or skills.

npx --yes github:jpolvora/workflow-skills

Auto-Update

Update installed skills. Preserves shared/config.json, stack.md, MEMORY.md, and memory/.

npx --yes github:jpolvora/workflow-skills update
npx --yes github:jpolvora/workflow-skills update --include-new

Non-interactive install

For agents/CI. Overwrites skills; always preserves consumer-owned shared/ files. Non-TTY requires --yes.

npx --yes github:jpolvora/workflow-skills install --full --yes
npx --yes github:jpolvora/workflow-skills install --package workflows --yes
npx --yes github:jpolvora/workflow-skills install --skills spec-to-pr,08-fix-pr --yes

Fresh fetch / cache

Never append @latest or @main to the github: specifier (exits 128). Bust cache by clearing npx cache, then re-run with npx --yes.

npx --yes github:jpolvora/workflow-skills update

Version Check

Compare installed vs latest online, or print the local version.

npx --yes github:jpolvora/workflow-skills --check
npx --yes github:jpolvora/workflow-skills --version

cURL (shim → npx)

Same CLI as npx. Requires Node.js. Pass args after bash -s --.

curl -fsSL https://raw.githubusercontent.com/jpolvora/workflow-skills/main/install-skills.sh | bash -s --
curl -fsSL https://raw.githubusercontent.com/jpolvora/workflow-skills/main/install-skills.sh | bash -s -- install --full --yes
curl -fsSL https://raw.githubusercontent.com/jpolvora/workflow-skills/main/install-skills.sh | bash -s -- update
curl -fsSL https://raw.githubusercontent.com/jpolvora/workflow-skills/main/install-skills.sh | bash -s -- update --include-new
