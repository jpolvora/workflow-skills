## 0. Summary & Business Rules

**Feature / issue:** [#56](https://github.com/jpolvora/workflow-skills/issues/56) — Installer docs teach a broken `github:…@latest` form (npm exit 128); interactive install blocks agents/CI with per-skill overwrite prompts and unconditional `console.clear()`.

**Core objectives:**
1. Stop recommending `npx github:jpolvora/workflow-skills@latest` (and `@main` suffix) anywhere in-repo; document canonical `npx --yes github:jpolvora/workflow-skills` plus real cache-bust guidance and exit-128 troubleshooting.
2. Add non-interactive `install` flags so agents/CI can install Full / package / skill lists with `--yes` (overwrite skills, always preserve `config.json`).
3. Fix interactive UX: one-shot overwrite confirmation (or skip prompts under `--yes`); guard `console.clear()` with `process.stdout.isTTY`; refuse interactive overwrite loops when stdin is not a TTY unless `--yes`.
4. Keep `update` / `update --include-new` as the existing-consumer refresh path; align `install --yes` copy semantics with `runUpdate` (`copyDirPreservingConfig`).

**Business rules:**
- `shared/config.json` (and any skill-level `config.json`) is consumer-owned — never overwritten on install/update (`copyDirPreservingConfig` / hub ensure already enforce this).
- `self-learning/memory/` and `MEMORY.md` remain preserved (existing ignore rules; do not regress promote-shared memory traps).
- `assertNotSelfOverwrite()` unchanged.
- All user-facing CLI/docs strings: **en-us**.
- Optional npm registry publish so `npx workflow-skills@latest` works is **nice-to-have / out of must-ship** (AC1–AC5 do not require it).

**Security mitigations:**
- No new network or shell exec beyond existing CLI.
- Non-interactive overwrite only when `--yes` is explicit (no silent mass overwrite on bare `install` without flags in non-TTY).
- Self-overwrite guard remains.

**Complexity:** **simple–moderate** — primarily `bin/cli.js` + docs/tests; no skill FSM or dependency-graph redesign. Fits lite sequential exec (≤3 implementation steps, ≤2 layers: cli + tests/docs).

### CLI design (`bin/cli.js`)

**Current gaps:**
- `printHelp()` and `--check` newer-available message advertise `@latest`.
- `main()` only handles `update` vs default `runInteractive`.
- `runInteractive` always `console.clear()`; per-skill `Overwrite? (y/n)`.

**Proposed argv surface:**

```text
npx github:jpolvora/workflow-skills install --full --yes
npx github:jpolvora/workflow-skills install --package workflows --yes
npx github:jpolvora/workflow-skills install --skills spec-to-pr,08-fix-pr --yes
```

**Parsing (surgical):**
1. If `args[0] === 'install'`: parse flags; validate selection; require `--yes` when `!process.stdin.isTTY` (always recommend documenting `--yes` for CI); call new `runInstall(skills, { mode, packageKey?, skillNames?, yes })`.
2. Else keep existing `update` / flags / interactive default.

**`runInstall` behavior:**
1. `assertNotSelfOverwrite()`.
2. Build `selectedNames` via:
   - `--full` → `applyPackageSelection('full', …)`
   - `--package <key>` → `applyPackageSelection(key, …)` (validate key ∈ full|workflows|extra)
   - `--skills csv` → select listed names + `applyTransitiveDeps`; error on unknown names
3. `migrateRenamedSkills` / `migratePromotedSkills` (same as interactive).
4. For each selected skill: if dest exists and `yes` → `copyDirPreservingConfig`; if dest exists and interactive path → handled in interactive one-shot; if new → `copyDirSync`.
5. Hub ensure + `installPackagedAgentsIndex` (mirror interactive success path).

**Interactive overwrite (one-shot):**
- Before install loop, collect existing dests among selection.
- If any exist: single prompt `Overwrite N existing skill(s)? (y/n):` (list names optionally truncated).
- On yes: copy all with `copyDirPreservingConfig` (drop redundant double-write of config bytes if already preserved by helper — keep behavior correct, prefer calling helper once like `runUpdate`).
- If `!process.stdin.isTTY` and existing dests and no `--yes`: exit 1 with message pointing to `install … --yes`.

**TTY guard:** replace bare `console.clear()` with `if (process.stdout.isTTY) console.clear();`.

**Help / --check copy:**
- Remove `@latest` line.
- Document `install` flags + Troubleshooting bullet: exit 128 → do not use `@latest` on `github:`; use `npx --yes github:jpolvora/workflow-skills`.
- Cache-bust note: prefer clearing npx cache or re-running with `npx --yes`; optional `github:jpolvora/workflow-skills#main` only if verified locally during impl (do not document untested forms).
- `--check` “newer available” message → `npx --yes github:jpolvora/workflow-skills update` (no `@latest`).
