## Description

Combined upstream harness fixes from two related `check-harness` audits (2026-07-17): consumer install on cursor-reviewer (#65) and ERP.Fiscal.sync (#66). Deliver one PR on `workflow-skills` so the next consumer `update` does not reintroduce these problems.

### From #65 (broader)

1. **Phantom skill routes** in packaged `.agents/AGENTS.md` — index + Task router list skills not present on disk in that consumer install (or not shipped). Sync index with actual installable inventory; drift-check root hub + packaged index.
2. **Missing § Skill loading / § Precedence / opt-outs** — `gabarito`, `caveman`, `spec-format` cite sections that must exist on the packaged hub (and root hub as source of truth). Progressive disclosure: skills link; do not duplicate bodies.
3. **Unprefixed skill folder refs** in `spec-to-pr/STEP-DISPATCH.md` — Step 11 `integration-validation` → `07-integration-validation`; Step 13 `goal-fix-pr` → `09-goal-fix-pr`.
4. **Broken consumer link** `shared/AGENTS.md` → `../../../bin/skill-dependencies.json` (no `bin/` in consumer clones).
5. **Hub pointer** — document dual-hub: workflow routing → packaged `.agents/AGENTS.md`; consumer root may stay product-specific and should point at packaged index.

Out of scope (consumer-only): product `file:///` hubs, PT-BR product contracts, retired runner-local skill tables.

### From #66 (subset + extras)

1. Same STEP-DISPATCH Step 11 prefix fix (confirm exact `Task` `07-integration-validation` wording).
2. **Misleading link labels** in `spec-to-pr/SKILL.md` Post-12 line: label `code-review` / `fix-pr` → `06-code-review` / `08-fix-pr` (targets already correct).
3. Same `shared/AGENTS.md` consumer-safe dependency-map wording (prefer plain text + upstream repo link, not a broken relative path).

Consumer-only rows in #66 stay out of scope for this upstream PR.

### Suggested verification
